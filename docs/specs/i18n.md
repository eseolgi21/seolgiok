# seolgiok i18n 명세

## 아키텍처

| 항목 | 값 |
|---|---|
| 라이브러리 | next-intl ^4.12.0 |
| 라우팅 방식 | App Router `[locale]` 세그먼트 |
| 플러그인 진입점 | `./src/i18n/request.ts` (`next.config.ts`에서 `createNextIntlPlugin` 등록) |
| 로케일 감지 | 비활성화(`localeDetection: false`) — Accept-Language 헤더 무시, 쿠키는 proxy.ts에서 직접 처리 |
| 네비게이션 유틸 | `createNavigation(routing)` → `Link`, `useRouter`, `usePathname`, `redirect` 내보냄 |
| 메시지 로딩 방식 | 요청 시 `getRequestConfig` 서버 콜백에서 모든 네임스페이스 병렬 동적 import |

### 파일 역할

| 파일 | 역할 |
|---|---|
| `src/i18n/routing.ts` | 로케일 목록·기본 로케일 정의, 내비게이션 유틸 내보내기 |
| `src/i18n/request.ts` | 요청별 메시지 로더 — 로케일 정규화 + 네임스페이스 병렬 import |
| `next.config.ts` | `createNextIntlPlugin`으로 Next.js에 next-intl 통합 |

## 네임스페이스 목록

| # | 네임스페이스 | JSON 파일 | 사용 영역 |

| 3 | `authSignup` | `authSignup.json` | 회원가입 페이지 |
| 4 | `home` | `home.json` | 홈(랜딩) 페이지 전체 |
| 5 | `footer` | `footer.json` | 하단 푸터 |
| 6 | `location` | `location.json` | 오시는 길 페이지 |
| 7 | `menu` | `menu.json` | 메뉴판 페이지 |
| 8 | `announcement` | `announcement.json` | 공지사항 목록·상세 |
| 9 | `event` | `event.json` | 이벤트 목록·상세 |
| 10 | `help` | `help.json` | 고객센터·FAQ |

**총 12개 네임스페이스.**

## 키 구조 예시

### `header.json` — 계층형 객체

```json
{
  "public": { "menu": "메뉴판", "location": "오시는 길" },
  "app": { "dashboard": "대시보드", "myConfig": "개인 설정" },
  "auth": { "signup": "회원가입", "login": "로그인", "logout": "로그아웃" },
  "aria": { "home": "홈으로 이동", "myAccount": "내 계정" },
  "status": { "startFailed": "시작 실패" }
}
```

사용 예: `t('header.auth.login')`

### `authLogin.json` — 브랜드 네임스페이스 래핑

```json
{
  "Seolgiok": {
    "title": "로그인",
    "fields": { "idLabel": "아이디 또는 이메일" },
    "buttons": { "submit": "로그인" },
    "messages": { "successDesc": "{username}님, 환영합니다." }
  }
}
```

사용 예: `t('authLogin.Seolgiok.messages.successDesc', { username })`

### `authSignup.json` — 브랜드 래핑 없는 평면 계층

```json
{
  "title": "회원가입",
  "fields": { "username": "아이디", "email": "이메일" },
  "checklist": { "len": "8~18 문자 길이" },
  "agreements": { "terms": "이용약관에 동의합니다." },
  "buttons": { "submit": "회원가입" },
  "messages": { "successTitle": "회원가입 완료" },
  "errors": { "usernameTaken": "이미 사용 중인 아이디입니다." }
}
```

사용 예: `t('authSignup.errors.usernameTaken')`

### `location.json` — 단순 + 중첩 혼합

```json
{
  "title": "오시는 길",
  "hours": { "weekday": "월 – 금", "weekdayTime": "09:00 – 익일 09:00" },
  "parking": { "desc": "건물 지하 주차장 이용 가능" },
  "faq": { "qAddress": "설기옥은 어디에 있나요?" },
  "opening": { "dDay": "오픈까지", "days": "일" }
}
```

### `account.json` — 기능 단위 구분

```json
{
  "page": { "title": "내 계정" },
  "otp": { "registerBtn": "OTP 등록", "registerSuccess": "OTP가 성공적으로 등록되었습니다." },
  "wallet": { "inputLabel": "EVM 지갑 주소 (0x...)", "saveSuccess": "지갑 주소가 등록되었습니다." },
  "password": { "changeBtn": "비밀번호 변경" },
  "toast": { "copySuccess": "복사 완료" },
  "error": { "loadFailed": "프로필 조회 실패 (HTTP {status})" }
}
```

## 워크플로우

### 새 UI 문자열 추가 절차

1. **네임스페이스 결정**: 관련 기능 페이지에 대응하는 JSON 파일 선택 (없으면 `request.ts`의 `NAMESPACES`에 추가)
2. **ko 파일에 키 추가**: `src/i18n/messages/ko/<namespace>.json`에 한국어 값 먼저 작성
3. **5개 로케일 동시 추가**: `en`, `ja`, `vi`, `zh` 동일 경로 파일에 같은 키 구조로 번역 추가
4. **컴포넌트에서 사용**: `const t = useTranslations('<namespace>')` → `t('<group.key>')`

```tsx
// 예시: authLogin 네임스페이스
const t = useTranslations('authLogin');
// <p>{t('Seolgiok.messages.successDesc', { username })}</p>
```

### 5개 로케일 동시 추가 규칙

- **원자성**: ko만 추가하고 나머지를 나중에 추가하는 것은 금지
- **키 구조 동일**: 모든 로케일에서 JSON 키 구조(경로)가 완전히 일치해야 함
- **값만 번역**: 키 이름·계층 구조는 로케일 무관 동일

### 새 네임스페이스 추가 절차

1. `src/i18n/messages/ko/<newNamespace>.json` 파일 생성 (한국어 기준)
2. `src/i18n/messages/en|ja|vi|zh/<newNamespace>.json` 생성 (동일 구조, 각 언어 번역)
3. `src/i18n/request.ts`의 `NAMESPACES` 배열에 `"newNamespace"` 추가
4. 빌드(`npm run build`) 또는 개발 서버(`npm run dev`) 재시작으로 확인

### 누락 키 감지 방법

next-intl은 런타임에서 존재하지 않는 키 접근 시 콘솔에 경고를 출력한다.

```
[next-intl] Could not resolve: namespace.missing.key
```

- **수동 비교**: `ko/` 파일 키 목록을 `en|ja|vi|zh/`와 diff로 비교
  ```bash
  # 예: ko vs en 비교 (header.json)
  diff <(jq -r 'paths | join(".")' src/i18n/messages/ko/header.json | sort) \
       <(jq -r 'paths | join(".")' src/i18n/messages/en/header.json | sort)
  ```
- **빌드 에러**: `importMessages` 함수에서 파일 로드 실패 시 `i18n load error: locale=..., ns=...` 에러 throw → 빌드 중단

## 절대 규칙

| 규칙 | 내용 |
|---|---|
| **하드코딩 금지** | UI에 노출되는 모든 문자열은 반드시 next-intl 키로 제공 (`"로그인"` 직접 삽입 불가) |
| **5개 로케일 동시 추가** | 새 키는 ko·en·ja·zh·vi 5개 파일에 **동시에** 추가 (하나라도 누락 시 런타임 에러) |
| **키 구조 동기화** | 모든 로케일의 JSON 키 경로가 100% 일치해야 함 |
| **i18n 변경 경유** | 번역 키 추가·수정·로케일 라우팅 변경은 반드시 `i18n-expert` 에이전트 경유 |
| **`request.ts` NAMESPACES 등록** | 새 JSON 파일 생성 시 반드시 `NAMESPACES` 배열에도 등록 (미등록 시 메시지 로드 안 됨) |
| **KRW 금액 하드코딩 금지** | 금액 표시도 i18n 키로 처리하거나 인터폴레이션 사용 |
